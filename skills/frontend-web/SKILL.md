# Frontend Web

Use this skill when changing user-visible web UI, frontend application behavior, styling, or component structure.

## Workflow

1. Inspect the existing component, routing, styling, and design conventions before editing.
2. Keep the change scoped to the requested workflow or screen.
3. Preserve accessibility basics: semantic controls, labels, keyboard operation, focus states, and readable contrast.
4. Verify the rendered UI in a browser when the change is visible.
5. Capture visual evidence for PRs when the issue or Ralph loop requires it.

## Verification

- Run the relevant unit, type, lint, or build command for the frontend surface.
- Use a browser check for layout, interaction, and responsive behavior when practical.
- Include screenshot or other visual evidence for UI-visible changes.
