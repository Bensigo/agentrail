# Landing v2 ("Jace") Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the public landing page (`apps/console/app/(marketing)/`) as an 8-section persona-led story — hero with live phone demo, sticky-stack use cases, named-step lemon band, channels scene, live cached stats, top-up billing — per the owner-approved 2026-07-22 design.

**Architecture:** The page stays a server component composed of small client scene components in `(marketing)/`. The existing drift-guarded `ConversationDemo` survives and is re-hosted inside a hero phone frame driven by a timer instead of scroll. Stats come from a new `run_outcomes` count query exposed through `unstable_cache` (1 h) and a public `GET /api/v1/stats`. Owner-supplied mascot renders become the brand marks.

**Tech Stack:** Next.js 15 App Router (RSC + client islands), Tailwind arbitrary-value classes over the TASTE.md CSS custom properties, vitest raw-source craft pins, Drizzle/Postgres via `@agentrail/db-postgres`.

## Global Constraints

- Tokens only: colors via `var(--gray-*)`, `var(--accent-*)`, `var(--paper)`; **no new hex** outside `LIGHT_SURFACE` (page.tsx).
- Lemon `--accent-fill` is fill-with-dark-text ONLY (`--accent-fill-text`); lemon never text on light. (craft pin)
- Weights: 400/700 only — no `font-medium/semibold/extrabold`. (craft pin)
- Type sizes: `text-heading-1/2`, `text-label`, `text-body-sm`, `text-mono-data`, default body — no ad-hoc `text-[..px]`. (craft pin)
- Fonts: Source Serif 4 display via `(marketing)/layout.tsx` only; Inter body; Berkeley Mono data. No new font loads. (craft pin)
- Rendered em-dashes in `page.tsx` ≤ 2. (craft pin)
- "X, not Y" rebuttal lines ≤ 2 page-wide (audit CO-3): exactly `Nothing merges without you.` and `didn't land — counted, not hidden`.
- Typing-cursor motif ≤ 2 (hero display line, closing headline).
- Sub-14px text on `--paper` uses `--gray-11` or darker (audit GQ-1).
- Motion: `transform`/`opacity` only, ease-out family, click-triggered reveals < 300 ms; scenes degrade to static on `prefers-reduced-motion` and mobile fallbacks.
- The Approve tap in the demo is NEVER auto-fired (controller ruling) — visitor clicks it.
- Owner rulings 2026-07-22 (record in TASTE.md): channels presented all-available (Telegram/Slack/Discord equal; non-live buttons route to sign-in, never dead links); billing = pay-for-what-you-use top-up model; mascot canon = 4 renders.
- PR discipline: 3 stacked PRs (`feat/landing-v2-hero` → `feat/landing-v2-story` → `feat/landing-v2-numbers`), each green on `pnpm --filter @agentrail/console test` + `typecheck`.

### Task 8: `countRunOutcomes()` (db-postgres)
- [ ] In `packages/db-postgres/src/queries/run_outcomes.ts`:
```ts
export interface RunOutcomeCounts { success: number; humanReview: number; failed: number; }
export async function countRunOutcomes(): Promise<RunOutcomeCounts> {
  const rows = await db
    .select({ outcome: runOutcomes.outcome, n: sql<number>`count(*)::int` })
    .from(runOutcomes)
    .groupBy(runOutcomes.outcome);
  const byOutcome = Object.fromEntries(rows.map((r) => [r.outcome, r.n]));
  return {
    success: byOutcome["success"] ?? 0,
    humanReview: byOutcome["human_review"] ?? 0,
    failed: byOutcome["failed"] ?? 0,
  };
}
```
(match the file's existing `db`/`sql` import style; export through the same barrel `recordRunOutcome` uses.)

### Task 9: `lib/landing-stats.ts` + tests + route
- [ ] ```ts
/** Documented dogfood record — docs/benchmarks/results/dogfood-track-record.md. */
export const DOGFOOD_BASELINE = { workedOn: 53, shipped: 33, didntLand: 20 } as const;
export interface LandingStats { workedOn: number; shipped: number; didntLand: number; source: "live+baseline" | "baseline-only"; }
export async function computeLandingStats(count: () => Promise<RunOutcomeCounts> = countRunOutcomes): Promise<LandingStats> {
  try {
    const c = await count();
    return { workedOn: DOGFOOD_BASELINE.workedOn + c.success + c.humanReview + c.failed,
             shipped: DOGFOOD_BASELINE.shipped + c.success,
             didntLand: DOGFOOD_BASELINE.didntLand + c.humanReview + c.failed,
             source: "live+baseline" };
  } catch { return { ...DOGFOOD_BASELINE, source: "baseline-only" }; }
}
export const getLandingStats = unstable_cache(() => computeLandingStats(), ["landing-stats"], { revalidate: 3600 });
```
