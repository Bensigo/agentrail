This walks through the fastest path from nothing to a reviewed pull request:
sign up, connect GitHub, and message Jace. No CLI, no runner to install —
the work runs in the cloud. (Want to run the worker on your own machine
instead? See [Self-hosting](/docs/getting-started/self-hosting).)

## 1. Sign up

Go to [heyjace.com](https://heyjace.com) and sign in with GitHub. The first
time in, you'll create a workspace — just a name and a slug.

## 2. Connect GitHub

On the setup steps, connect GitHub: add the repo(s) you want Jace to work in.
The workspace creates the GitHub webhook automatically, so issues can flow
into your queue. If that fails, it gives you a payload URL and secret to add
by hand.

## 3. Message Jace on Telegram

Connect a channel: Telegram is the one open door today. Message the bot at
the link shown once — that conversation becomes your connection to Jace, no
token to paste.

If you're on a self-hosted deploy with no shared bot configured, you'll see a
form to bring your own bot instead — see
[Self-hosting](/docs/getting-started/self-hosting).

## 4. Describe the work

Send Jace a message describing what you want done — a bug, a small feature,
anything scoped enough to hand to an engineer. You can also just hand over a
GitHub issue you've already written.

## 5. Approve the brief

Before touching any code, Jace replies with a brief: the task type, the model
it picked, and a dollar estimate. Approving is the one confirmation this flow
needs — it's what sets the run's budget.

## 6. Get your PR

Jace opens a pull request against your repo and messages you when it's ready.
You review and merge it — or turn on merge permission under **Settings →
Permissions** and Jace will merge it itself once its review gate is green.
