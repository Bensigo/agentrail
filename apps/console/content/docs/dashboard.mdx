The CLI works fully without an account — everything in
[Getting Started](/docs/getting-started/quickstart) runs on your machine alone.
The hosted dashboard is the **optional team layer**: it adds a shared view,
governance, and cross-repo cost on top of the same CLI. This page covers it as
the secondary path.

## Sign in

```bash
agentrail login
```

`login` signs this machine into an AgentRail workspace. Use `--url BASE_URL` to
point at a self-hosted server. Check who you're signed in as with `whoami`, and
sign out with `logout`:

```bash
agentrail whoami
agentrail logout
```

## Queue issues in the console

Once signed in, queue work from the console — the team-facing dashboard. Open it
with:

```bash
agentrail console
```

Queued issues enter the workspace **Issue Queue**, each carrying its tier (which
model), remaining budget, and state — and each leaving in exactly one
[Run Outcome terminal](/docs/concepts/review-and-gates): Green,
Escalated-to-human, or Blocked.

## Run the local worker

```bash
agentrail runner --concurrency 2
```

`runner` is the local worker that pulls queued issues from the workspace and
executes them on your machine. The work happens locally — your keys, your
sandbox — while the workspace coordinates the queue.

| Flag | Description |
|------|-------------|
| `--idle SECONDS` | How long to wait for work before idling out. |
| `--once` | Process one batch, then exit. |
| `--concurrency N` | Number of issues to run in parallel. |

## What the team layer adds

The CLI is one developer's terminal. The console is the team system:

- **Every developer's runs in one workspace** — context packs, cost, gate
  evidence, and outcomes from all your runs in one place.
- **Cost across repos and teams** — real-dollar cost rolled up beyond a single
  laptop, so spend never surprises you.
- **A source-linked audit trail** — who or what performed each sensitive action.
- **Members and roles** — invite teammates and govern who can configure review
  gates and manage repositories.

Indexing still stays local on each developer's machine; the console stores run
metadata, context-pack citations, costs, and audit events — not your source by
default.
