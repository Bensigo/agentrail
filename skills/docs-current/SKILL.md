# Docs Current

Use this skill when the task depends on current external documentation, third-party APIs, package behavior, licensing, source provenance, or rapidly changing platform rules.

## Workflow

1. Prefer official docs, source repositories, release notes, or package metadata over memory.
2. Record the source URL, branch or version, and observed date when provenance matters.
3. Treat third-party examples and community repositories as references, not trusted install sources.
4. Check license and audit status before copying or vendoring external content.
5. Separate verified facts from inferences in implementation notes and PR evidence.

## Verification

- Include the docs or source link used for any current external claim.
- Re-run relevant tests or checks after applying current-doc guidance.
